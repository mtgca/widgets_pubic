Backpropagation Through Time: The Multiplicative Challenge

To train an RNN, the loss gradient must travel backwards through the unrolled chain — and every step back multiplies it by the same shared weight matrix WhhT. Repeated multiplication by one matrix is numerically unstable: the gradient explodes or vanishes depending on the largest eigenvalue λmax of Whh. Drag the sliders to see both failure modes.

Where the multiplication comes from

1 · The recurrence

ht= tanh( Wxhxt + Whhht−1 )

The same matrix Whh connects every hidden state to the next — parameter sharing across time.

2 · Chain rule over time

∂ℒ∂h0 = ( ∏t=1T ∂ht∂ht−1 ) ∂ℒ∂hT

Reaching back T steps chains T Jacobians — each one contains a factor of WhhT.

3 · The norm bound

‖∂ℒ∂h0‖ ≲ λmaxT ⋅ ‖∂ℒ∂hT‖

λmaxT is a geometric sequence: it races to ∞ when λmax>1 and collapses to 0 when λmax<1.

💥 Fixing exploding gradients

Gradient clipping: if the gradient norm exceeds a threshold, rescale it back down before the update.

Crude but effective — it keeps one bad batch from producing NaN weights and crashing training.

🔗 Fixing vanishing gradients

Gated architectures (LSTM, GRU): an additive cell-state path with gates lets gradients flow back without repeated multiplication by Whh.

Careful initialization (e.g. orthogonal Whh) and ReLU variants also help; attention/Transformers sidestep recurrence entirely.

⚖️ Why there is no easy middle

Keeping λmax=1 exactly is a knife-edge — training constantly changes Whh, and the tanh derivative (≤ 1) shrinks the product further.

That is why plain RNNs struggle with long-range dependencies even when carefully tuned.
