RNNs: Hidden-State & Output Update — How the Matrix Shapes Fit

At every time step an RNN performs two matrix multiplications: ht=tanh(WhhTht−1+WxhTxt) and y^t=WhyTht. Pick the sizes below and watch the real matrices. Hover any result entry to see exactly which row and which vector produce it — every output number is one dot product.

1 · Choose the three sizes

Everything in the two equations is built from just three numbers.

n — input dim length of xt 3
d — hidden dim length of ht 4
k — output dim length of y^t 2

2 · Hidden state update maps input space + previous hidden state → hidden space

ht= tanh( WhhT ht−1 + WxhT xt )

3 · Output vector maps hidden space → output space

y^t= WhyT ht
Bias terms bh∈Rd×1 and by∈Rk×1 omitted for clarity — each would simply be added to its column before the activation.

Why the transpose?

Wxh is stored as n×d, but to turn an n-vector into a d-vector we need a d×n matrix — so we multiply by WxhT.

Many textbooks define the weight matrix already transposed and write Wxt — same mathematics, different storage convention.

Inner sizes must match

In (a×b)·(b×1) the two inner b's touch and cancel; the result is a×1.

Columns = size of the vector the matrix can consume; rows = size of the vector it produces.

How many parameters?

Grow d and watch Whh grow quadratically — the hidden-to-hidden matrix usually dominates.

Notation follows the lecture slide: input xt∈Rn×1, hidden state ht∈Rd×1, output y^t∈Rk×1.
