2Two ways to compute the exact same xt

An embedding layer is not a different kind of math — it is a linear layer (weights E, no bias, no activation) applied to a one-hot input. Because the input has a single 1 and zeros everywhere else, the full matrix multiplication collapses to reading one row. Compare both computations for the selected token :

Key insight: both paths use the same learned parameters (the |V|×d matrix E), produce the same output, and receive the same gradients during training — only row i of E gets a nonzero gradient for this token, because every other row was multiplied by 0. The embedding layer is simply the efficient implementation: it skips building the one-hot vector and skips every multiplication by zero.

⚠ Why not feed the index i straight into a linear layer?

A raw index is a categorical label, not a quantity. Feeding it as a number would tell the network that dog (1) is "between" cat (0) and fish (2), and that zebra (5) is "five times" dog — pure artifacts of alphabetical ordering. One-hot encoding (and therefore the embedding lookup) removes this fake ordinal structure: every token starts equally distant from every other.

Why no bias term?

A bias b would add the same vector to every token's embedding: xt=E[i]+b. Since each row of E is free to learn any values, the rows can absorb any constant shift — the bias adds parameters without adding expressive power. That is why nn.Embedding has none.
