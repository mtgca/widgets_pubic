Transposed Convolutions & the Checkerboard Artifact

A convolution you already know, run in reverse — and why its overlaps can paint a checkerboard.

1From gather to scatter

A regular convolution gathers: each output pixel reads a k×k window of the input and reduces it to one number. A transposed convolution scatters: each input pixel takes the whole kernel, scales it by its own value, and stamps that k×k patch into the output. Where stamps overlap, they sum.

That flip also reverses the size arithmetic. With stride s and no padding, a transposed convolution grows the input:

o=(i−1)·s+k

Left: convolution, one shaded 3×3 window → one output cell. Right: transposed convolution, one input cell → one shaded 3×3 stamp in the output. Same wiring, arrows reversed — which is why it is called transposed: it multiplies by the transpose of the convolution's matrix. It is not the inverse of a convolution (a common misconception), just a learnable upsampler with conv-like weight sharing.

With s = 2 and k = 3, stamps land every 2 pixels but are 3 pixels wide — so some output cells receive 1 stamp and their neighbors receive 2 or 4. Turn on show overlap counts and stamp all: the counts alternate. That alternation is the seed of the checkerboard.

3It's just a convolution in disguise

If you prefer the convolution you already know: a transposed convolution with stride s gives the same result as (1) inserting s−1 zeros between input pixels, (2) padding the border with k−1 zeros, and (3) running a normal stride-1 convolution over it. Hover an input pixel to find it in the dilated grid.

Now the checkerboard has a second explanation: as the stride-1 kernel slides over this grid, some window positions cover 4 real pixels, others cover 2 or 1 — the rest of each window is inserted zeros. Adjacent output cells are computed from different amounts of real signal.

Strictly, the equivalent kernel is the original rotated 180°; the kernel used here is symmetric, so it makes no difference.

4The checkerboard, isolated

Strip away the values: feed in an image of all 1s with a kernel of all 1s, so every output cell simply counts how many stamps reached it. If that count is uneven in the interior, no learned kernel can fully hide it — the multiplicity pattern is baked into the architecture. Explore k and s:

Borders always receive fewer stamps — that is the ordinary edge effect, not the checkerboard. The artifact is the alternating pattern in the interior, which appears exactly when k mod s ≠ 0. When k=s, coverage is uniform because stamps tile perfectly with no overlap at all.

5How practitioners avoid it

Same experiment — uniform input, count the coverage — for three ways to upsample ×2:

The resize-then-convolve fix (nearest or bilinear upsampling followed by a plain stride-1 convolution) separates upsampling from filtering, so no overlap arithmetic can go wrong. It is the standard remedy proposed by Odena, Dumoulin & Olah (2016), “Deconvolution and Checkerboard Artifacts” (Distill). Choosing k divisible by s helps, but uneven gradients can still re-introduce faint patterns during training — resize-conv is the safer default.
