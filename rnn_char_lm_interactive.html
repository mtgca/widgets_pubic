RNNs: Character-Level Language Modeling

Train a tiny recurrent network to predict the next character of the word “hello” — then use it to generate text, and discover what the temperature knob really does.

The toy setup

Four characters are enough to see everything a real language model does.

Vocabulary V=[h,e,l,o]  ·  |V|=4
Input / output dim n=4,  k=4 (match the vocabulary)
Hidden state dim d=3 (the latent representation)
Training pairs from “hello” h → e e → l l → l l → o

Training — predictive learning with one-hot representations

Each character enters as a one-hot vector; the network must give the highest score (logit) to the character that comes next. Click a time step to follow it.

Focus
Parameter sharing: the same three weight matrices Wxh, Whh, Why process the entire sequence — every column of the diagram reuses them.

Input encoding

Each character is a one-hot vector — a single 1 at its vocabulary index.

Multiplying Wxhxt then simply selects one column of Wxh.

State recurrence

ht=tanh(Whhht−1+Wxhxt)

The hidden state mixes the current input with everything read so far — that is why “l” can be followed by “l” once and “o” the next time.

Objective

y^t=Whyht gives 4 logits. Training maximizes the logit at the target character’s index (green) and pushes the others (red) down.

Inference — autoregressive generation through a feedback loop

Once trained, the model generates on its own: logits → Softmax → probability distribution → pick a character → feed it back as the next input. Click a time step to follow it.

Focus

Argmax — greedy decoding

Always pick the character with the highest probability.

Deterministic: the same start character always generates exactly the same text.

Sampling

Draw the next character at random, using the softmax probabilities as the odds.

Stochastic: mostly picks likely characters, but can surprise you — and temperature controls how often. ↓

Temperature — the randomness dial

Before Softmax, every logit is divided by a constant T. That one division is the whole trick — drag the slider and watch the distribution reshape while the ranking never changes.

Softmax(zi) = ezi/T ∑jezj/T

This is exactly the “temperature” hyperparameter of modern LLMs

A large language model is the same machine scaled up: a next-token predictor with a vocabulary of ≈100000 tokens instead of 4 characters. When you call an LLM API, the temperature you pass divides its logits exactly as the slider above:

response = client.messages.create(
    model="claude-sonnet-5",
    temperature=0.2,   # <-- the same T you just explored
    messages=[...]
)
  • T ≈ 0 – 0.3 — near-greedy, repeatable answers: code, math, data extraction.
  • T ≈ 0.7 – 1.0 — the usual chat default: fluent but varied phrasing.
  • T > 1 — flatter distribution: brainstorming and creative writing; push it too far and low-probability tokens get real mass — the text degrades into incoherence.
  • T = 0 in an API is greedy decoding: the argmax token every time.
