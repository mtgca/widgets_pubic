The Core of Attention: A Library Analogy (Q, K, V)

You walk into a library with a question. Every book shows a label on its spine (the Key — used only for matching) and holds content inside (the Value — what you actually take away). Attention is the librarian: it compares your question (the Query) against every label, and hands you a mixture of contents, weighted by how well each label matched. Pick a query below and follow the four steps.

1 The Query (Q) — what are you looking for?

Your contextual need — the question you bring to the library. In attention, every word brings its own question: “what information do I need from the other words?”

2 The Library — Keys are matched, Values wait inside

The librarian compares the query against every spine label and turns the matches into attention weights that always add up to 100%.

KEY spine label · visible · used for matching VALUE content inside · retrieved, never matched
The separation is the point: the Key decides how much attention a book gets; the Value is what actually flows out. Matching and content are two different jobs, so attention keeps a separate Key and Value for every book on purpose.

3 The Retrieval — a weighted blend of Values

No book is “selected” and none is discarded. Every Value flows into the result — each contributing exactly its attention weight.

4 The Result — a context-aware representation

The blend depends entirely on the query: change the question and the same library returns a different mixture.

Attention blends — it does not pick a single winner. The result is a mixture of Values: mostly the best match, but with a little of everything else. Nothing is ever fully ignored, so the model can draw on several relevant sources at once.
