The Core of Attention: A Library Analogy (Q, K, V)

You walk into a library with a question. Every book shows a label on its spine (the Key — used only for matching) and holds content inside (the Value — what you actually take away). Attention is the librarian: it compares your question (the Query) against every label, and hands you a mixture of contents, weighted by how well each label matched. Pick a query below and follow the four steps.

1 The Query (Q) — what are you looking for?

Your contextual need. In a transformer this is a vector the current token computes to ask: “what information do I need from the others?”

2 The Library — Keys are matched, Values wait inside

The matching engine compares the query against every spine label and softmax turns the match scores into attention weights that always sum to 100%.

KEY spine label · visible · used for matching VALUE content inside · retrieved, never matched
The separation is the point: the Key decides how much attention a book gets; the Value is what actually flows out. A book can advertise one thing and contain another — matching and content are two different jobs, so attention learns two different vectors for them.

3 The Retrieval — a weighted sum of Values

No book is “selected” and none is discarded. Every Value flows into the result — each contributing exactly its attention weight.

4 The Result — a context-aware representation

The blend depends entirely on the query: change the question and the same library returns a different mixture.

Attention is a weighted average, not an if/else. The output is a mixture of Values — mostly the best match, but with a little of everything else. That softness is what lets gradients flow and lets the model hedge between several relevant sources.

One thing the analogy hides: in self-attention there is no separate reader — every token plays all three roles at once. It asks a question (Q), advertises a label (K), and offers content (V), all computed from its own embedding by three learned matrices WQ,WK,WV. Every token visits the library, and every token is also a book.

This rescales the scores before softmax — the same role the 1/dk factor plays in the formula. Sharper → the winner takes (almost) all; softer → weights spread toward a uniform average. Watch the bars in steps 2–4 react.
